# 仕様: 完了タスク数カウンタの配置見直し

- 状態: 確定
- 関連: [`../../planning/backlog.md`](../../planning/backlog.md) BL-047
- 依存: BL-035 (ui-redesign-foundation / 配置未確定の明示) / BL-008 (completion-counter / 実装本体)
- 由来要件: FR-040 (今日の完了タスク数を表示)
- 関連 NFR: NFR-001 (単一ワークフロー強制) / NFR-010 (最小手数の起票) / NFR-013 (並び順の予測可能性)

## 背景 / 課題

BL-008 (completion-counter) で実装された「今日の完了タスク数」は、現在 `today-view.tsx` の
起票フォーム直上に `<div aria-label="今日の完了タスク数"><span>今日の完了: N</span></div>` として
配置されている。

BL-035 (ui-redesign-foundation) では UI 全面刷新のモックアップを確定させたが、そのモックアップに
完了数カウンタに対応する要素が存在しなかった。このため BL-035 の spec.md に以下が記録された。

> REQ-8: 完了数カウンタはモックに表示されていない（FR-040 は必須仕様のため別の配置を BL-047 で確定する）
> U-003: 候補は (a) サイドバー下部、(b) focus-view 内統合、(c) today-view 見出し右。
>         保守側デフォルト案: (c) today-view 見出しの右に小さく出す。

その後 BL-036〜BL-046 でサイドバー・AppShell・各ビューのデザイントークン置換が完了した。
この段階で完了数カウンタは「起票フォームより前の独立した div」として残っており、
刷新後の UI 構造とのズレが生じている。具体的には以下の問題がある。

1. **ヘッダ領域との分離**: `<header>` には h1（「今日」）と「＋プロジェクトの追加」ボタンが
   既に共存している。完了数は同じヘッダ文脈で読むと意味的に自然だが、現状は header の外に浮いている。
2. **文脈の不一致**: 完了数は「今日のタスク」の文脈で示す情報であり、今日ビュー以外の画面
   （focus-view / tomorrow-view）では参照不要である。サイドバーへの移動は常時表示となり冗長。
3. **スタイルの未整備**: BL-046 のデザイントークンが整備されたが、カウンタ要素には
   `TODO(BL-046)` マーカーが付いておらず、トークン参照が未導入のままである。

本 BL では配置の確定とスタイルの整備を行い、BL-008 の既存テストを green のまま維持する。

## ゴール / 非ゴール

### ゴール

- 完了数カウンタを `today-view.tsx` の `<header>` 内、h1 と「＋プロジェクトの追加」ボタンの間に
  移動し、見出し右領域に小さく表示する。
- カウンタ要素にデザイントークンを適用したスタイルを追加し、視覚的に適切な表示にする。
- BL-008 の受け入れ基準（`aria-label="今日の完了タスク数"` を持つ要素に数値が表示される）を
  引き続き満たす。
- 既存の全テスト（単体・E2E）が green を維持する。

### 非ゴール

- **完了数カウンタの機能変更**: `completionCount` の取得・インクリメント・リセット処理は
  BL-008 で確定済み。本 BL では UI 配置の変更のみを行い、サーバ API・ドメイン・Repository には
  一切触れない。
- **サイドバーへの移動**: BL-035 U-003 の候補 (a) は採用しない。完了数は「今日のタスク」
  文脈の情報であり、全ページ共通のサイドバーに常時表示するのは設計原則（NFR-001 単一ワークフロー）
  に反する過剰露出になる。
- **focus-view への統合**: 候補 (b) は採用しない。focus-view は「現在のタスク」に集中する
  単一責務の画面であり（BL-037 spec REQ-2）、今日の進捗集計を表示するのは責務外。
- **カウンタの ON/OFF 設定**: NFR-012（設定項目最小化）に反するため提供しない。
- **「今日の完了:」以外の文言変更**: BL-008 のテストが文言を検証しているため本 BL では変更しない。

## 要件

### 機能要件

- **REQ-1 配置**: 完了数カウンタを `today-view.tsx` の `<header>` 内に移動する。
  h1（「今日」）の右側、「＋プロジェクトの追加」ボタンの左側に配置する。
  ヘッダの 3 要素の並びは「h1（左）| カウンタ（中）| ＋プロジェクトの追加（右）」とする。

- **REQ-2 マークアップ**: 要素は `<span aria-label="今日の完了タスク数">今日の完了: {N}</span>`
  として、BL-008 のテストが参照する `aria-label` を維持する。
  ラッパー `<div aria-label="...">` は削除し、インライン表示に適した `<span>` に変更する。

- **REQ-3 スタイル**: `today-view.css`（存在しない場合は新規作成）にカウンタのスタイルを追加する。
  `--font-size-small` / `--color-fg-subtle` のデザイントークンを使い、
  ヘッダ内で控えめに（本文より小さく、グレー系で）表示する。

- **REQ-4 today-view 以外での非表示**: 完了数カウンタは today-view のヘッダにのみ表示する。
  focus-view / tomorrow-view / settings-view 等には表示しない（現状維持）。

### 非機能要件

- **NFR-001（単一ワークフロー）との整合**: カウンタは参照のみで、操作 UI（補正ボタン等）を
  一切持たない。表示 1 箇所・設定なし・サーバ正本値反映の方針を維持する。
- **NFR-010（最小手数の起票）との整合**: カウンタを header に統合することで、起票フォームと
  カウンタが視覚的に分離される。起票の手順に変更はない。
- **BL-008 互換性**: `aria-label="今日の完了タスク数"` の識別子でカウンタを取得しているテストが
  引き続き green であること。ただし `aria-label` が `<div>` ではなく `<span>` に移ることは許容する。
