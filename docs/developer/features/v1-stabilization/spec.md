- 状態: 確定
- 関連: [`../../planning/backlog.md`](../../planning/backlog.md) の BL-024

## 背景 / 課題

BL-001〜BL-023 の全実装が完了し、531 件のテストが通過している。
しかし「実装が完了している」と「公開可能な状態である」は別の基準である。
v1.0.0 としてリリースタグを打つには、次の点を客観的に確認・記録しておく必要がある。

- 実装済みの機能が FR-001〜FR-070（機能要件）と NFR-001〜NFR-050（非機能要件）を漏れなく満たしているか。
- バックログの全アイテムが Done になっているか。
- テストが全件 green であることが記録されているか。
- リポジトリが OSS として公開可能な状態にあるか（BL-022 完了確認）。
- プロジェクト成功基準 SC-001「1 か月以上日常的に使い続けられる」を満たすための手動確認が済んでいるか。

## ゴール / 非ゴール

- ゴール:
  - 要件カバレッジ監査ドキュメントを作成し、全 FR・NFR に対する適合状況を一覧で示す。
  - テスト通過記録（件数・実行日時）を残す。
  - バックログ BL-001〜BL-023 が全件 Done であることを確認し記録する。
  - GitHub リポジトリ公開に必要なファイルの存在を最終確認する。
  - ドッグフーディング完了を手動チェックリストで確認できる状態にする。
  - 上記が全て完了したときに `v1.0.0` タグを打つ条件を定義する。
- 非ゴール:
  - 新しい機能の追加・変更は行わない。
  - ドッグフーディング期間の自動計測・強制は行わない（手動確認に留める）。
  - 既存のテストコードや実装コードの変更は原則行わない。
