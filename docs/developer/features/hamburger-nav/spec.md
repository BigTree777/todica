# 仕様: ハンバーガーナビゲーション

- 状態: 確定
- 関連: [`../../planning/backlog.md`](../../planning/backlog.md) の BL-049

## 背景 / 課題

現在の `AppShell` は幅 200px の固定サイドバーを常時表示するレイアウトを採用している。
このサイドバーはメイン領域の幅を常に圧迫しており、特にモバイル幅での表示体験が悪い。
ユーザーの要望により、ハンバーガーボタンで開閉するオーバーレイメニューに変更する。

## ゴール / 非ゴール

- ゴール:
  - 常時表示の固定サイドバーを廃止し、ハンバーガーボタンで開閉するオーバーレイメニューに置き換える
  - メイン領域 (`.app-shell__main`) が画面幅を全幅利用できるようにする
  - ナビゲーションリンクの構成（プライマリ 3 件 + セカンダリ 4 件）は現状を維持する
  - アクセシビリティ要件（キーボード操作・スクリーンリーダー対応）を満たす
- 非ゴール:
  - ナビゲーション項目の追加・削除・並び順変更
  - レスポンシブブレークポイントによる自動開閉切り替え（デスクトップは常時表示、モバイルはハンバーガー、等）
  - アニメーションの詳細な演出仕様の追求（動作することが最低条件）

## 要件

- 機能要件:
  - REQ-1: 画面左上に常時表示のハンバーガーボタン（`☰`）を配置する
  - REQ-2: ハンバーガーボタンをクリックするとオーバーレイメニューが開く
  - REQ-3: メニューが開いている状態で、メニュー外の領域をクリックするとメニューが閉じる
  - REQ-4: メニューが開いている状態で、ナビゲーションリンクを選択するとメニューが閉じる
  - REQ-5: メニューが開いている状態で、`Escape` キーを押すとメニューが閉じる
  - REQ-6: メニューが閉じているとき、`.app-shell__main` が全幅（100%）を占有する
  - REQ-7: ナビゲーションリンクの構成は既存を維持する（プライマリ 3 件: 現在のタスク・今日のタスク・明日のタスク、区切り線、セカンダリ 4 件: プロジェクト・ルーティン・ゴミ箱・設定）
  - REQ-8: アクティブなリンクには視覚的なアクティブ表示を維持する
- 非機能要件:
  - REQ-9: ハンバーガーボタンに `aria-label="メニューを開く"` / `aria-label="メニューを閉じる"` を状態に応じて付与する
  - REQ-10: ハンバーガーボタンに `aria-expanded` 属性（`true` / `false`）を状態に応じて付与する
  - REQ-11: オーバーレイメニューに `aria-modal="true"` および `role="dialog"` を付与する
  - REQ-12: メニューを開いたとき、フォーカスをメニュー内の最初のリンクに移動する
  - REQ-13: メニューが閉じたとき、フォーカスをハンバーガーボタンに戻す
  - REQ-14: スタイルはデザイントークン（`tokens.css`）の既存変数のみを使用する

## 受け入れ基準

### AC-1: ハンバーガーボタンの初期表示

```
シナリオ: ページロード時にハンバーガーボタンが表示される
  Given アプリが起動している
  When  任意のページ（/today など）を開く
  Then  画面左上にハンバーガーボタン（☰）が表示される
  And   aria-expanded="false" が付与されている
  And   aria-label="メニューを開く" が付与されている
  And   オーバーレイメニューは表示されていない
```

### AC-2: メニューを開く

```
シナリオ: ハンバーガーボタンをクリックするとメニューが開く
  Given ページが表示されていてオーバーレイメニューが閉じている
  When  ハンバーガーボタンをクリックする
  Then  オーバーレイメニューが表示される
  And   ボタンの aria-expanded="true" に変わる
  And   ボタンの aria-label="メニューを閉じる" に変わる
  And   メニュー内の最初のリンク（現在のタスク）にフォーカスが移動する
```

### AC-3: リンク選択でメニューが閉じる

```
シナリオ: ナビゲーションリンクを選択するとメニューが閉じる
  Given オーバーレイメニューが開いている
  When  メニュー内のナビゲーションリンク（例: 今日のタスク）をクリックする
  Then  対応するビューに遷移する
  And   オーバーレイメニューが閉じる
  And   ハンバーガーボタンの aria-expanded="false" に戻る
```

### AC-4: メニュー外クリックで閉じる

```
シナリオ: メニュー外の領域をクリックするとメニューが閉じる
  Given オーバーレイメニューが開いている
  When  オーバーレイ背景（メニューパネル外の暗転領域）をクリックする
  Then  オーバーレイメニューが閉じる
  And   ハンバーガーボタンにフォーカスが戻る
```

### AC-5: Escape キーで閉じる

```
シナリオ: Escape キーを押すとメニューが閉じる
  Given オーバーレイメニューが開いている
  When  Escape キーを押す
  Then  オーバーレイメニューが閉じる
  And   ハンバーガーボタンにフォーカスが戻る
```

### AC-6: メイン領域の全幅表示

```
シナリオ: メニューが閉じているときメイン領域が全幅を占有する
  Given オーバーレイメニューが閉じている
  When  今日のタスク画面を表示する
  Then  .app-shell__main の幅が 100% である（固定サイドバーが存在しない）
```

### AC-7: ナビゲーション項目の構成

```
シナリオ: メニュー内にすべてのナビゲーション項目が表示される
  Given ページが表示されていてオーバーレイメニューが閉じている
  When  ハンバーガーボタンをクリックしてメニューを開く
  Then  プライマリナビとして「現在のタスク」「今日のタスク」「明日のタスク」が表示される
  And   区切り線が表示される
  And   セカンダリナビとして「プロジェクト」「ルーティン」「ゴミ箱」「設定」が表示される
```

### AC-8: アクティブリンクの表示

```
シナリオ: 現在表示中のページのリンクがアクティブ表示になる
  Given /today を表示している
  When  ハンバーガーボタンをクリックしてメニューを開く
  Then  「今日のタスク」リンクにアクティブスタイル（太字など）が適用されている
```

### AC-9: アクセシビリティ属性

```
シナリオ: オーバーレイメニューに適切な ARIA 属性が付与される
  Given ページが表示されている
  When  ハンバーガーボタンをクリックしてメニューを開く
  Then  メニューパネルに role="dialog" が付与されている
  And   メニューパネルに aria-modal="true" が付与されている
  And   メニューパネルに aria-label または aria-labelledby が付与されている
```
