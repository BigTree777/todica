## ADR-0001: 永続化に PostgreSQL を採用する（雛形流用. 履歴として残す）

> 本 ADR はテンプレ流用時の記入例として作成されたもの. 当初の決定根拠（単一ユーザー・ローカル前提）は誤りであったため, 過去に ADR-0002 で廃止した. その後, 構成が「自前サーバを伴うクライアント・サーバ型」に修正されたため, サーバ DB の本決定は ADR-0007（サーバ側技術スタック）で改めて行う.

- 状態: 廃止（→ [ADR-0002](0002-supersede-postgresql-adoption.md)）
- 日付: 2025-01-15（廃止: 2026-06-05）
- 決定者: プロジェクトオーナー（個人開発）

## 背景 / 状況

- 雛形コピー時の記入例として「永続化先を決める必要があった」想定で残されていた.
- ADR-0002 段階での廃止理由は「単一ユーザー・端末ローカル前提なのでサーバ側 RDBMS は不要」というものだったが, この前提自体が誤りであったことが後に判明している（自前サーバを伴う構成に修正済み. ADR-0006 参照）.
- それでも本 ADR の決定文面（雛形のままの記入例）は再利用できないため, 状態は「廃止」のまま維持する.
- サーバ側で PostgreSQL を採用するか否かは, ADR-0007（サーバ側技術スタック）で改めて検討する.

## 検討した選択肢

| 選択肢 | 利点 | 欠点 |
| --- | --- | --- |
| A: <RDBMS> | 整合性・実績 | スケール時の手間 |
| B: <NoSQL> | 柔軟・スケール | 整合性の担保が難しい |

## 決定

（廃止）当初は A（PostgreSQL）採用と記載されていたが, 本 ADR は ADR-0002 により廃止する. サーバ側で PostgreSQL を採用するかどうかの本決定は [ADR-0007](0007-server-tech-stack.md) を参照のこと.

## 結果 / 影響

- 良い影響: なし（廃止）.
- トレードオフ / 注意点: 本 ADR の決定（雛形のままの記入例）は採用していない. 引用しないこと. PostgreSQL の採否そのものは ADR-0007 で別途判断する.
