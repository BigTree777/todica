## ADR-0002: ADR-0001（PostgreSQL 採用）を廃止する

> ADR-0001 は雛形コピー段階で作成された記入例であり, 当時の前提（ローカル限定）と整合しないため廃止した. その後, 構成前提が「自前サーバを伴うクライアント・サーバ型」に修正されたため, 本 ADR の廃止理由のうち「サーバを置かないので RDBMS は不要」という記述は前提崩れとなっている. ただし「ADR-0001 の決定文面（雛形そのまま）を採用しない」という結論自体は依然として有効なため, 本 ADR は **承認のまま維持** し, サーバ側で PostgreSQL を採用するか否かは [ADR-0007](0007-server-tech-stack.md) で改めて扱う.

- 状態: 承認（ADR-0001 を Supersede. ただし廃止理由の一部は ADR-0006 / ADR-0007 で前提が更新されている）
- 日付: 2026-06-05
- 決定者: プロジェクトオーナー（個人開発）

## 背景 / 状況

- `docs/developer/adr/0001-use-postgresql.md` はテンプレ流用時の記入例として「永続化に PostgreSQL を採用する」という決定を残している.
- 雛形のままの「PostgreSQL 採用」決定文面は, 根拠や代替案が記入例のままで Todica の検討に使えない. このため決定文面ごと廃止する.
- 本 ADR の当初廃止理由には「単一ユーザー・端末ローカル前提なのでサーバ DB は不要」という記述があったが, この前提は誤りで, 構成は **自前サーバを伴うクライアント・サーバ型** に修正されている（ADR-0006）.
- それでも「雛形のままの ADR-0001 を採用しない」という結論は依然として有効なため, 本 ADR は承認のまま維持する.
- サーバ側で PostgreSQL を採用するか否かは ADR-0007（サーバ側技術スタック）で改めて扱う. Android のローカルモードの永続化は ADR-0009（Android クライアント技術スタック）で扱う.

## 検討した選択肢

| 選択肢 | 利点 | 欠点 |
| --- | --- | --- |
| A: ADR-0001 を Superseded に変更し, 本 ADR-0002 で廃止理由を記録する | 履歴として「なぜ PostgreSQL ではないのか」が残り, 後から誤って引き直されない | ADR-0001 を残すぶん, 一度確認する手間が残る |
| B: ADR-0001 を削除する | ノイズが減る | 「過去にそういう案があった」という履歴が消え, 同じ議論を蒸し返しやすい |

## 決定

A を採用する.

- `0001-use-postgresql.md` の状態を `廃止（→ ADR-0002）` に書き換え, 本 ADR-0002 を「ADR-0001 を廃止する決定」として残す.
- サーバ側で PostgreSQL を採用するか否かの本決定は ADR-0007 で行う.

## 結果 / 影響

- 良い影響: テンプレ起源の決定が現要件と衝突したまま放置されることを防ぐ. ADR 履歴の連続性が保たれる.
- トレードオフ / 注意点: 本 ADR の廃止理由のうち「サーバを置かないので RDBMS は不要」という記述は ADR-0006（配布構成の再定義）により前提崩れになっている. PostgreSQL の採否そのものは ADR-0007 で改めて判断する.
