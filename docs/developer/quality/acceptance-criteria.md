# 受け入れ基準（Acceptance Criteria）

> 「完成」を判定するための基準の書き方。各機能の spec からこの形式を参照する。

## 書き方の原則

- 検証可能（Yes/No で判定できる）であること
- 実装方法ではなく「満たすべき結果」を書くこと

## 形式: Given / When / Then

```
シナリオ: <名称>
  Given <前提条件>
  When  <操作>
  Then  <期待される結果>
```

### 例

```
シナリオ: 正しい資格情報でログインできる
  Given 登録済みユーザーが存在する
  When  正しいメールとパスワードでログインする
  Then  ダッシュボードに遷移する
```

## チェックリスト形式（補助）

- [ ] <満たすべき条件 1>
- [ ] <満たすべき条件 2>
- [ ] 異常系: <エラー時の期待挙動>
