## リリース計画

> SemVer に準拠。v0.x はプレリリースで破壊的変更を許容し, v1.0.0 で要件カバレッジ完了 + Play Store 公開とする。

| バージョン | 予定日 | 主な内容 |
| --- | --- | --- |
| v0.1.0 | 2026-Q3 | **最小タスク管理（Web + サーバ）.** タスク CRUD（FR-001, FR-002, FR-005, FR-009）, 優先度（FR-003, FR-004）, 完了 / 削除アクション（FR-006, FR-007）, 2 階層固定（FR-008）, 今日ビュー（FR-010, FR-011, FR-014）, 現在のタスク / 自動繰上げ（FR-012, FR-013）, 完了数カウント（FR-040）, 境界時刻・リセット（FR-041, FR-042, FR-043, FR-051）, ゴミ箱（FR-060, FR-061, FR-062）, 設定（境界時刻のみ）. NFR-001 / NFR-002 / NFR-010〜NFR-013 / NFR-020 / NFR-021 を満たす. プロジェクト・ルーティンは未提供（タスクは全て単独タスク）. Android なし. |
| v0.2.0 | 2026-Q4 | **プロジェクト + ルーティン追加.** プロジェクト管理（FR-020, FR-021, FR-022）, プロジェクト所属タスクの今日ビュー横断表示, ルーティン定義（FR-030, FR-035）, 指定曜日の自動生成（FR-031）, 翌日に持ち越さない挙動（FR-033）, 履歴・ストリーク非保持（FR-034）の確認. PM ツール非連携（FR-070）の明示. |
| v0.3.0 | 2027-Q1 | **PWA インストール + オフライン書込キュー.** Service Worker でシェル pre-cache, `manifest.webmanifest` 提供（NFR-031 の Android 化前提）, IndexedDB ベースの書込キュー + Background Sync, `Idempotency-Key` ヘッダ必須化, `If-Match` 楽観ロック, 競合時の選択 UI. オフライン耐性（architecture §7.5）の完成. |
| v0.4.0 | 2027-Q1 | **Android（Capacitor サーバモード）.** Capacitor プロジェクト化, Web 実装の共有, サーバモードで自前サーバに接続. Android の起動時 / 設定でのモード選択（ローカル UI のみ. ローカル実体は v0.5.0）, Google Play Internal Testing への提出フロー確立（NFR-030, NFR-031 の検証, RISK-002 への一次対応）. |
| v0.5.0 | 2027-Q2 | **Android ローカルモード.** `@capacitor-community/sqlite` による端末内永続化, ローカルモード時のリセット処理（アプリ起動時 / フォアグラウンド境界時刻到達時の両経路）, サーバモード / ローカルモードのデータ独立性確認. ADR-0011 の T3 ハイブリッドを両モードで実機検証. |
| v1.0.0 | 2027-Q2 | **全要件カバー + 安定化 + Play Store 公開.** FR-001〜FR-070 / NFR-001〜NFR-050 のカバレッジ最終確認, NFR-050（OSS 公開: ライセンス・依存関係・秘密情報の取扱い）整備, NFR-030（Play Store 規約・審査）対応, 1 か月以上のドッグフーディング（SC-001）を経て Google Play Store 正式公開. |

## 今は対象外（将来検討）

> project.md §8 Out of Scope に挙げられた項目は全期間を通じて対象外。本ロードマップでは, それに加えて当面着手しない事項を明示する。

- 進捗管理 / 進捗率・進捗バー（OOS-001）.
- プロジェクトのアーカイブ（FR-021, OOS-002）.
- サブタスク / サブプロジェクト / ネスト（FR-008, OOS-003）.
- 検索機能（OOS-004）.
- "保留" その他のユーザー定義ステータス・カスタムフィールド・タグ・ラベル（OOS-005）.
- ビュー切替（カンバン / ガント / カレンダー / テーブル）, 保存フィルタ, ユーザー定義ソート（OOS-006）.
- 開始日, 3 日以降の期限指定, （通常タスクの）繰り返し, リマインダー通知（OOS-007, FR-002 で値域として保証済み）.
- ルーティンの実施履歴 / ストリーク（FR-034, OOS-008）.
- 成果物そのもののファイル管理・保存（OOS-009）.
- 外部連携 / 自動化ルール / カスタマイズ設定画面（FR-070, OOS-010, NFR-012 と整合）.
- 複数ユーザー / チーム機能, 共有 SaaS としてのホスティング（NFR-002, NFR-021, OOS-011, OOS-012）.
- サーバモードとローカルモードの間のデータ移送・同期（architecture §2 / §3, NFR-021 関連の将来検討事項）.
- 共有 SaaS としての外部 API 公開（architecture §9, ADR-0010）.
