# クイックスタート

> Todica を最短で使い始めるための手順。

Todica は自分でサーバを立てて使う「サーバ接続モード」と、Android アプリで端末内にデータを持つ「ローカルモード」のいずれかで使えます。

## 1. サーバ接続モード（Web / Android 共通）

ブラウザ（PWA）からでも Android アプリからでも、共通して 1 台のサーバに接続して使う構成です。複数端末から同じデータを見たい場合はこちらを選びます。

### 1-1. サーバを立てる

セルフホストが前提です。サーバの構築手順は [`docs/developer/setup/server.md`](../developer/setup/server.md) を参照してください。起動時に `AUTH_TOKEN` 環境変数で認証トークンを設定します。

### 1-2. Web クライアントから接続する

1. サーバの URL（例: `http://localhost:3000`）をブラウザで開きます。
2. 初回アクセス時に「サーバ接続設定」画面が出るので、サーバ URL と `AUTH_TOKEN` を入力します。
3. 「接続する」を押すと今日のタスク一覧（最初は空）が表示されます。

PWA としてホーム画面に追加すると、アプリのように起動できます（オフライン時もキャッシュから起動でき、書き込みは後でサーバに同期されます）。

### 1-3. Android アプリから接続する

Google Play からインストール（内部テスト中）したのち、初回起動時の「サーバ接続設定」画面でサーバ URL と `AUTH_TOKEN` を入力します。

## 2. ローカルモード（Android のみ）

サーバを立てずに、Android 端末の中だけにデータを持つ使い方です。1 台で完結するシンプルな構成で、他端末との同期はできません。

1. Android アプリを初回起動します。
2. 「サーバ接続設定」画面で「ローカルモードで使う」を選びます。

以降はサーバ接続を介さず、端末内のデータベースに直接読み書きします。あとからサーバ接続モードに切り替えることもできます（設定画面から）。

## 3. 最初のタスクを作る

1. プロジェクトを作る（任意。単発タスクのみでも可）。
2. タスクを足す（期限は既定で今日、優先度は最優先 / 普通 / 後回しから選択）。
3. 今日のタスクから 1 つを選んで「現在のタスク」にすると、それだけが大きく表示されます。
4. 完了アクションで消化すると、自動で次のタスクに進みます。
